Where's my done-flag?

Oozie workflows work great for pipelining the ETL from a single data source. When a process depends on multiple data sources, it might be time to bump up your game a notch and bust out the Oozie coordinators.

With data pipelining via multiple coordinator output events, one can aggregate ouput events in a consuming coordinator via multiple input events.

Working with a coordinator that consumes multiple input events, I found that certain actions (hive, shell, maybe more...) failed to produce the standard _SUCCESS done-flag on completion.

Turns out, the way to create this flag is by adding another action in your workflow.

I used a shell action to write the flag to the output directory. This is less than ideal, as it requires configuration changes (output directory) in multiple locations. If anyone has suggestions, I'm all ears.

The other limitation with this method is that it will only create one _SUCCESS flag, for the current day. I haven't got Oozie to send configuration properties from the coordinator to the workflow yet, but when that is working, one could substitute the date in createSuccessFlag.sh with the proper, Oozie defined params.

coordinator.xml

<?xml version="1.0" encoding="UTF-8"?>
<coordinator-app xmlns="uri:oozie:coordinator:0.4" name="import-somesource" frequency="${coord:days(1)}" start="2014-09-03T00:01Z" end="2050-01-01T00:01Z" timezone="UTC">
  <controls>
    <!-- Wait 23 hours before giving up -->
    <timeout>1380</timeout>
    <concurrency>1</concurrency>
    <execution>LIFO</execution>
  </controls>
  <datasets>
    <dataset name="somesource-load" frequency="${coord:days(1)}" initial-instance="2014-09-03T08:00Z" timezone="America/Los_Angeles">
      <uri-template>${dataPath}/someapp/${YEAR}-${MONTH}/${DAY}/data</uri-template>
    </dataset>
  </datasets>
  <output-events>
    <data-out name="output" dataset="somesource-load">
      <instance>${coord:current(0)}</instance>
    </data-out>
  </output-events>
  <action>
    <workflow>
      <app-path>${workflowApplicationPath}</app-path>
    </workflow>
  </action>
</coordinator-app>

workflow.xml

<?xml version="1.0" encoding="UTF-8"?>
<workflow-app xmlns="uri:oozie:workflow:0.4" name="someapp-wf">
  <start to="shell-import"/>
  <action name="shell-import">
    <shell xmlns="uri:oozie:shell-action:0.2">
      <job-tracker>${jobTracker}</job-tracker>
      <name-node>${nameNode}</name-node>
      <exec>${importFile}</exec>
      <file>${importFile}#${importFile}</file>
    </shell>
    <ok to="shell-create-success-flag"/>
    <error to="email-fail"/>
  </action>
  <action name="shell-create-success-flag">
    <shell xmlns="uri:oozie:shell-action:0.2">
      <job-tracker>${jobTracker}</job-tracker>
      <name-node>${nameNode}</name-node>
      <exec>${createSuccessFlag}</exec>
      <file>${createSuccessFlag}#${createSuccessFlag}</file>
    </shell>
    <ok to="email-success"/>
    <error to="email-fail"/>
  </action>
  <action name="email-fail">
    <email xmlns="uri:oozie:email-action:0.1">
      <!-- ... secret email config goes here -->
    </email>
    <ok to="fail"/>
    <error to="fail"/>
  </action>
  <action name="email-success">
    <email xmlns="uri:oozie:email-action:0.1">
      <!-- ... secret email config goes here -->
    </email>
    <ok to="end"/>
    <error to="end"/>
  </action>
  <kill name="fail">
    <message>Hive failed, error message[${wf:errorMessage(wf:lastErrorNode())}]</message>
  </kill>
  <end name="end"/>
</workflow-app>

job.properties

hadoopMaster=hadoop-master.mydomain.com
nameNode=hdfs://${hadoopMaster}:8020
jobTracker=${hadoopMaster}:8050
tzOffset=-8
oozie.use.system.libpath=true
oozie.libpath=/user/oozie/share/lib
appPath=${nameNode}/apps
dataPath=${appPath}/data

workflowApplicationPath=${appPath}/someapp
oozie.coord.application.path=${workflowApplicationPath}
#oozie.wf.application.path=${workflowApplicationPath}

importFile=import.sh
createSuccessFlag=createSuccessFlag.sh

createSuccessFlag.sh

#!/bin/bash

set -o nounset  # exit if trying to use an uninitialized var
set -o errexit  # exit if any program fails
set -o pipefail # exit if any program in a pipeline fails, also
set -x          # debug mode

YEAR=`date +"%Y"`
MONTH=`date +"%m"`
DAY=`date +"%d"`
HDFS_PATH=hdfs://hadoop-master.mydomain.com:8020/apps/data/someapp/$YEAR-$MONTH/$DAY/data

HADOOP_USER_NAME=hdfs hadoop fs -mkdir -p $HDFS_PATH
HADOOP_USER_NAME=hdfs hadoop fs -touchz $HDFS_PATH/_SUCCESS
